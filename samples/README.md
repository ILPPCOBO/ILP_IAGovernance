# Sample policy packages (illustrative only)

These files are **illustrative, preliminary mock outputs** generated by
`scripts/generate-samples.ts`. They exist so you can see the shape of what the
rules engine produces without filling in the questionnaire by hand.

> ⚠ **Not legal advice.** These samples are preliminary drafts for
> informational purposes only. They do **not** constitute legal advice, are
> **not** a final compliance review, and do **not** create a lawyer–client
> relationship. Any AI policy must be reviewed and adapted by qualified
> professionals before implementation. The companies named below are fictional.

## What's here

| File | Scenario | Language | Readiness |
| --- | --- | --- | --- |
| `mature-en.json` / `mature-en.en.md` | **Northwind Analytics** — a 51–200-person professional-services firm with full governance roles, broad human review, an incident process, vendor review, training and enterprise-tier tools. | English | ~91/100 — *mature* |
| `early-es.json` / `early-es.es.md` | **Estudio Lumen** — a 1–10-person early-stage marketing studio using public/free ChatGPT (personal accounts) with confidential and client data and almost no governance. | Spanish | ~8/100 — *early stage* |

For each scenario:

- `<name>.json` is the full `PolicyPackage` object (carries **both** EN and ES
  text, so it can be rendered in either language).
- `<name>.<lang>.md` is `renderMarkdown(pkg, lang, "full")` — the full package
  rendered to Markdown in that scenario's primary language.

## Regenerating

```bash
npx tsx scripts/generate-samples.ts
```

The script logs each package's readiness score and finding count. Exact scores
and IDs can shift if the seed data or scoring rules change.
